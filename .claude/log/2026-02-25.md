# 2026-02-25 작업 로그

## Q: 여포 패시브 지속 물리공격력 증가 중복 안됨 확인

사용자: 여포의 패시브는 다른 이름이 다른 패시브와 다르게 이름은 다르지만 같은 지속 물리공격력 증가와 중첩되지 않다는게 확인되었어

### 수정 내용
- **문제**: 여포 패시브 FoolhardyBravery(용이무모)가 별도 필드로 Atk_Rate와 합산되어 중첩 적용됨
- **확인 사항**: 인게임에서 이름이 달라도 같은 "지속 물리공격력 증가" 카테고리로 MaxMerge됨
- **수정**:
  - `CharacterDB.cs`: 여포 패시브 `FoolhardyBravery = 31` → `Atk_Rate = 31`
  - `BuffSet.cs`: `FoolhardyBravery` 필드 및 모든 관련 코드(Add/MaxMerge/Clone/Clear/implicit operator) 제거
  - `StatCalculator.cs`: `+ partyPermanentBuff.FoolhardyBravery` 합산 제거
  - `MainWindow.xaml.cs`: `+ totalBuffs.FoolhardyBravery` 합산 제거
- **결과**: 여포 공격력 31%가 다른 지속 물리공격력 증가와 MaxMerge 처리됨
